Initializes the server structure.

Exemples:

libanio_init(&server, server_fd, 0, 21, &on_accept, &on_read, &on_eof, &on_error, ANIO_MODE_STREAM);

libanio_init(&server, server_fd, 14, 20, &on_accept, &on_read, &on_eof, NULL, ANIO_MODE_BLOCK, 1024);

libanio_init(&server, server_fd, 100, 100, &on_accept, &on_read, &on_eof, &on_error, ANIO_MODE_LINE, "\r\n");

Parameters
serverPointer to the t_anio structure to initialize.
fdFile description on already listening socket.
max_clientsMaximum simultaneously-connected clients allowed.
thread_pool_sizeMaximum number of threads to perform i/o operations and callbacks when needed.
fptr_on_acceptFunction pointer to be called when accepting a new client.
fptr_on_readFunction pointer to be called when data is read. The data's readiness to be consumed depends on the mode set on initialization: stream, block or line. Must not be NULL.
fptr_on_eofFunction pointer to be called when reaching end-of-file.
fptr_on_errorFunction pointer to be called when a syscall fails.
servermode Defines when to call the fptr_on_read callback (ANIO_STREAM: whenever data is read, ANIO_BLOCK: when a complete block is available, ANIO_LINE: when a complete line is available). Once set, the mode shall not be changed.
...Depending on the mode: ANIO_STREAM: nothing (ignored if any given), ANIO_BLOCK: block size, ANIO_LINE: array of chars representing the newline sequence.
Returns
(0) on success
(-1) on error
